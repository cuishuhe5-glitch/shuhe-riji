# 小黑日报助手走查与书赫日报助手优化空间

走查时间：2026-06-15
竞品版本：小黑日报助手 v1.3.1 mac-arm64
走查方式：Computer Use 实际点击界面，未登录、未授权、未提交任何外部操作。

## 已观察到的竞品结构

- 主导航：今日工作、生成报告、工作时间线、时段热力图、应用记录、历史报告、接入 Agent、订阅、邀请激励、隐私保护、设置、帮助、客服。
- 生成报告：报告类型、日期范围、模板卡片、右侧模板预览、自定义指令、开始生成报告。
- 模板体系：云端模板 + 内置模板，覆盖成果导向、会议驱动、AI 工作轨迹、三句话、老板一分钟、TOP3、项目推进、开发者、产品、运营、设计、管理者等场景。
- 今日工作：顶部价值主张、隐私承诺、空状态说明、时段热力、显示器选择卡片。
- 工作时间线：日期范围、导出数据、添加记录、搜索、核心指标、分类时长图表、复制日志、近 30 分/1 小时/2 小时/今天快捷筛选。
- 设置：语言、开机自启动、自动记录、截图间隔、截图保留、自动暂停、回车快速记录、单次识别当前屏幕、AI 分析方式、识别提示词管理、导入/导出/清除数据、木鱼小功能、Dock 图标、记忆、每日定时生成、版本日志、检查更新。
- 隐私保护：以信任叙事为主，解释截图即删、本地存储、直连分析、智能暂停、数据生命周期、采集范围、API Key 安全、第三方服务、FAQ。
- 接入 Agent：显示本地 HTTP 服务状态、接入地址、请求日志、一键复制 Agent/Skill 接入配置文本，强调每次先拉取实时 API 文档。

## 书赫日报助手的优先优化方向

### P0：Agent 接入页产品化

当前书赫日报助手已有 Agent/项目上下文能力，但竞品的 Agent 页更适合直接给其他 Agent 使用。

建议：
- 新增本地 API 文档入口，例如 `/api/docs` 或 `/api/agent-docs`，返回 Markdown。
- Agent 页展示完整可复制接入说明，包含服务地址、强制规则、接口发现方式、响应格式、示例请求。
- 增加“请求日志”入口，便于确认外部 Agent 是否真的调用过。

### P0：报告模板体系扩展

竞品最强的第一印象是模板多、场景明确、右侧实时预览。

建议：
- 把当前模板扩充为职业/场景模板库：成果导向、会议驱动、老板一分钟、TOP3、三句话、项目推进、开发者、产品、运营、设计、管理者。
- 报告页改成“左侧模板卡片 + 右侧模板预览”的结构。
- 每个模板显示适用人群、输出风格、是否内置/自定义。

### P1：隐私保护页改成信任说明

书赫日报助手已有隐私功能，但表达偏工程状态。

建议：
- 顶部使用明确承诺：截图即删、本地存储、直连模型、Key 安全。
- 增加数据生命周期图：截图 -> AI 分析 -> 保存文字记录 -> 删除截图。
- 增加 FAQ：截图是否保存、数据是否上传、Key 存在哪里、如何停用记录、如何迁移数据。

### P1：设置页补齐记录控制

竞品设置页把常用控制集中得更完整。

建议：
- 增加截图间隔设置。
- 增加鼠标闲置自动暂停时长。
- 增加“识别当前屏幕”单次记录按钮。
- 增加识别提示词管理入口。
- 增加 Dock 图标/桌面窗口模式说明。

### P1：历史报告筛选增强

竞品历史报告页更适合大量报告检索。

建议：
- 增加类型筛选：全部、日报、周报、月报。
- 增加快捷日期：本周、本月、最近 7 天、最近 30 天。
- 搜索框与刷新按钮保持在列表顶部。

### P2：时间线效率工具

竞品时间线有几项很实用的小能力。

建议：
- 增加“复制日志到剪切板”。
- 增加近 30 分/1 小时/2 小时/今天快捷筛选。
- 分类时长图支持柱状/饼图切换。

### P2：空状态和第一屏文案

竞品在无数据时解释“后台正在记录，开始工作后稍等即可看到”，降低用户焦虑。

建议：
- 今日概览空状态更明确说明：是否已启动记录、是否缺权限、何时会出现第一条记录。
- 第一屏加入更像产品的三句承诺：本地、隐私、自动生成日报。

### P3：轻量情绪化功能

竞品右下角“今日已敲 0 下 · 点击解压 · 右键静音”不是核心功能，但能增加记忆点。

建议：
- 可选，不优先。书赫日报助手更适合走工作工具路线，不一定需要照搬。

## 建议落地顺序

1. Agent 接入页产品化。
2. 报告模板库与右侧预览。
3. 隐私页信任说明。
4. 设置页补齐截图间隔、闲置暂停、单次识别、提示词管理。
5. 历史报告和时间线的筛选/复制效率工具。
